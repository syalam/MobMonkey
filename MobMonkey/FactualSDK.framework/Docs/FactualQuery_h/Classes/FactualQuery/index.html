Introduction

Encapsulates all the parameters supported by the Factual Query API

Discussion

To query a Factual table, you must first construct a Query object and specify some basic row selection criteria. You can optionally construct an empty Query object, which will return to you the first (limit) number of rows using a table's default sort order. There are some restrictions on what combinations of filters are valid in certain circumstances. Read the property / method descriptions for more details

includeRowCount

@property (nonatomic,
    assign) BOOL includeRowCount;
Discussion

when true, the response will include a count of the total number of rows in the table that conform to the request based on included filters. Requesting the row count will increase the time required to return a response. The default behavior is to NOT include a row count

limit

@property (nonatomic,
    assign) NSUInteger limit;
Discussion

used to limit the number of returned records in a single response. This system will return the lessor of either the limit value or the max limit value associated with the user's API Key. paginating a large record set.

maxValuesPerFacet

@property (nonatomic,
    assign) NSUInteger maxValuesPerFacet;
Discussion

The maximum number of unique facet values that can be returned for a single field. Range is 1-250. The default is 25.

minCountPerFacetValue

@property (nonatomic,
    assign) NSUInteger minCountPerFacetValue;
Discussion

For each facet value count, the minimum number of results it must have in order to be returned in the response. Must be zero or greater. The default is 1.

offset

@property (nonatomic,
    assign) NSUInteger offset;
Discussion

used to specify the offset (number of records to skip) when paginating a large record set.
